# MVP_SCOPE_v1 — Learning Contour (v1.1)

## 0. Назначение документа

Данный документ фиксирует **границы MVP Learning Contour** MatrixGin после завершения рефакторинга (Phases 1-5).

## 1. Определение MVP (v1.1)

**MVP Learning Contour** — это:
> Ограниченный контур MatrixGin, предназначенный для запуска обучения и наставничества сотрудников через **Dual-Interface (Web + Telegram)**, с обеспечением аналитической прозрачности и контроля целостности данных без карательных функций.

## 2. Основные Цели (v1.1)

1. Запустить обучение в режиме **Web-first для глубокого контента** и **Mobile-first (TG) для оперативных задач**.
2. Внедрить **Институт Наставничества** с прозрачной аккредитацией.
3. Обеспечить **Целостность Обучения** через пассивный Anti-Fraud мониторинг.
4. Сформировать доверие к системе через **Human-in-the-loop** аудит.

## 3. Функциональные границы (IN SCOPE)

### 3.1 Разрешённые функции
- **Web Course Player**: Полноценное прохождение курсов через браузер.
- **Quiz Engine**: Автоматизированная аттестация (Single/Multi-choice).
- **Trainer Dashboards**: Управление наставничеством и ротацией подопечных.
- **Security Dashboard**: Мониторинг аномалий и ручная валидация сигналов фрода.
- **Analytics Dashboards**: Визуализация Completion Rate и популярности треков.

### 3.2 Разрешённые интерфейсы
- **Telegram Bot**: Нотификации, статус и микро-обучение.
- **Web UI**: Образовательный портал и панели управления для HR/Admin/Trainer.

## 4. Жёсткие ограничения (OUT OF SCOPE)

### 4.1 Управление и контроль
- **Автоматические санкции**: Любые управленческие решения по результатам обучения требуют ручного подтверждения человеком.
- **KPI/Performance**: Результаты Университета не транслируются напрямую в KPI без участия руководителя.

## 5. MatrixCoin — режим MVP
MatrixCoin (MC) в v1.1 начисляется за **факт достижения квалификации** или завершения критических этапов наставничества. Система вознаграждений управляется через канонический Event Registry.

## 6. Критерии завершённости MVP
1. Стабильная работа **Anti-Fraud Engine** (без ложных блокировок).
2. Наличие **Full-Cycle** наставничества (Аккредитация -> Пробация -> Результат).
3. Доступность аналитики по всем активным академиям.
