## 1. Базовые методологические принципы: Андрагогика и Результат

В корпоративном обучении мы работаем со взрослыми людьми, имеющими опыт, устоявшиеся модели поведения и дефицит времени. Это требует отказа от школьной (педагогической) модели в пользу андрагогической.

### 1.1. Педагогика vs Андрагогика (Сравнительный анализ)

| Критерий | Педагогический подход (Школа/ВУЗ) | Андрагогический подход (Корпорация) |
| :--- | :--- | :--- |
| **Роль обучающегося** | Зависимый. Учитель определяет, что и как учить. | Независимый/Самоуправляемый. Партнер в обучении. |
| **Опыт** | Малоценен. Опыт учителя — основной ресурс. | Опыт ученика — богатый ресурс и база для обучения. |
| **Готовность учиться** | Определяется стандартной программой и возрастом. | Обусловлена потребностью решить рабочую проблему «здесь и сейчас». |
| **Ориентация** | Предметно-ориентированная (изучаем «Физику»). | Задаче-ориентированная (изучаем «Как починить станок»). |
| **Мотивация** | Внешняя (оценки, диплом, давление родителей). | Внутренняя (карьера, деньги, автономия, мастерство). |
| **Роль теории** | Теория первична, практика — иллюстрация. | Практика первична, теория — инструмент для решения. |

**Вывод:** Материалы корпоративного университета должны быть:
1.  **Прагматичными:** Отвечать на вопрос «Как это поможет мне работать лучше/зарабатывать больше?».
2.  **Уважающими опыт:** Не «разжевывать» очевидное, использовать опыт сотрудников как кейсы.
3.  **Модульными:** Позволять выборочное обучение (Just-in-Time Learning).

### 1.2. Outcomes-Based Education (OBE) — Обучение, основанное на результатах

Традиционный подход: *«Мы расскажем им про историю компании и свойства продукта»*.
OBE подход: *«Сотрудник должен уметь провести презентацию продукта за 3 минуты и закрыть сделку»*.

**Принцип:** Проектирование начинается «с конца» (Backward Design).
1.  **Бизнес-результат:** Увеличение продаж на 10%.
2.  **Поведенческий результат:** Сотрудники предлагают доп. услуги в 100% контактов.
3.  **Результат обучения:** Сотрудник *знает* матрицу кросс-сейла и *умеет* применять скрипт предложения.

### 1.3. Competency-Based Learning (CBL) — Компетентностный подход

Обучение измеряется не часами («прослушал 20 часов лекций»), а демонстрацией навыка.
*   **Знание (Knowledge):** «Знаю, что делать».
*   **Навык (Skill):** «Могу сделать».
*   **Компетенция (Competency):** «Делаю это регулярно, качественно и в разных условиях».

**Правило:** Модуль считается пройденным только когда сотрудник *продемонстрировал* компетенцию, а не просто сдал тест на запоминание.

## 2. Структура дисциплины

Архитектура дисциплины должна строиться по логике: **Цель → Оценка → Практика → Теория**.

### 2.1. Иерархия целей (Learning Objectives)
Используем таксономию Блума (в современной версии для бизнеса):

1.  **Уровень курса:** Глобальная цель.
    *   *Пример: «Самостоятельно вести сложные переговоры с VIP-клиентами».*
2.  **Уровень модуля:** Конкретный навык.
    *   *Пример: «Применять технику СПИН для выявления потребностей».*
3.  **Уровень урока:** Атомарное действие.
    *   *Пример: «Сформулировать 3 ситуационных вопроса для кейса Х».*

### 2.2. Логика модулей: Принцип «От простого к сложному» + «Scaffolding»
1.  **Why & What:** Зачем это нужно? Контекст. (Мотивация).
2.  **How (Algorithm):** Делай раз, делай два, делай три. (Демонстрация).
3.  **Try (Supported):** Попробуй сам с подсказками. (Тренажер).
4.  **Perform (Independent):** Сделай сам в реальной задаче. (Экзамен).

### 2.3. Связь теории с практикой: Модель 70:20:10
В структуре контента пропорции должны быть смещены:
*   **10% Теории:** Схемы, алгоритмы, принципы (Текст/Видео).
*   **20% Социального обучения:** Разбор полетов, обсуждение с ментором, форум.
*   **70% Практики:** Кейсы, симуляторы, рабочие задания (On-the-job training).

**Методическое требование:** На 1 единицу теории должно приходиться минимум 3 единицы практической отработки (тест, кейс, задание).

## 3. Рекомендации по созданию контента

### 3.1. Текстовые материалы
*   **Принцип:** «Пиши, сокращай». Корпоративный стиль — лаконичный, без воды.
*   **Структура:** Заголовки, буллиты, выделение жирным ключевых мыслей.
*   **Сторителлинг:** Сухая инструкция работает хуже, чем история: «Вася не проверил фланец, и вот что случилось...».

### 3.2. Визуальные материалы
*   **Схемы и инфографика:** Вместо 3 страниц текста — одна блок-схема процесса.
*   **Скриншоты/Фото:** Только реальные интерфейсы и оборудование. Никаких стоковых фото «счастливых людей в костюмах».
*   **Видео:** Микроформат (до 5-7 минут). 1 видео = 1 мысль/навык.

### 3.3. Практические задания и Кейсы
*   **Action Mapping (Cathy Moore):** Не спрашивайте «Что такое Х?». Спрашивайте «Клиент сказал Х, что вы ответите?».
*   **Реализм:** Кейсы должны быть взяты из реальной практики компании («В прошлый вторник на складе...»).
*   **Вариативность:** Правильный ответ не должен быть очевиден. Ошибки должны быть реалистичными (типичные заблуждения).

### 3.4. Тесты и Квизы
*   **Цель теста:** Не поймать на незнании, а закрепить знание.
*   **Дистракторы (неправильные ответы):** Должны быть правдоподобными.
*   **Обратная связь:** Самая важная часть теста.
    *   *Плохо:* «Неверно».
    *   *Хорошо:* «Неверно. Вы выбрали вариант А, но это приведет к потере клиента, так как... Правильно Б, потому что...».

## 4. Методика оценки знаний и навыков

### 4.1. Формативная оценка (В процессе)
*   **Цель:** Обучение и коррекция.
*   **Инструменты:** Встроенные квизы после каждого видео/раздела, самопроверка, диалоговые тренажеры.
*   **Влияние:** Не влияет на итоговый сертификат, показывает сотруднику его пробелы.

### 4.2. Суммативная оценка (Итоговая)
*   **Цель:** Проверка компетенции.
*   **Инструменты:** Итоговый тест, решение комплексного кейса, защита проекта, чек-лист наблюдения за работой.
*   **Критерии:** Pass/Fail (Сдал/Не сдал) по четкому порогу (например, 80% правильных действий).

### 4.3. Измеримость результата (Киркпатрик)
Уровень 2 (Learning) оцениваем тестами. Уровень 3 (Behavior) оцениваем через чек-листы на рабочем месте через 2-4 недели после обучения.

## 5. Усиление KPI и Экономическая эффективность

Дисциплина должна иметь **«Экономический паспорт»**:

1.  **KPI Link:** Явное указание, на какой метрике отразится обучение.
    *   *Дисциплина:* «Техники допродаж». *KPI:* Средний чек.
    *   *Дисциплина:* «Безопасность труда». *KPI:* Количество инцидентов/простоев.
2.  **ROI (Return on Investment):**
    *   Если обучение стоит дорого (разработка + время сотрудников), экономический эффект от изменения поведения должен перекрывать эти затраты.
3.  **Justification:** Если навык не влияет на деньги/риски — его не должно быть в корпоративном университете (это хобби).

## 6. Типовые ошибки (Anti-patterns)

1.  **Info-dump (Свалка информации):** «Давайте выложим все PDF-инструкции на портал». Это не обучение, это библиотека.
2.  **Talking Head (Говорящая голова):** Лектор читает слайды 40 минут. Внимание взрослого отключается на 6-й минуте.
3.  **Click-Next (Листалка):** Слайд-слайд-слайд-тест. Скучно, вызывает отторжение и «прокликивание».
4.  **Академизм:** Определения, история вопроса, теоретические споры. В бизнесе нужен алгоритм действия.
5.  **Отсутствие контекста:** «Делай так, потому что так надо». Взрослый требует ответа «Зачем?».

## 7. Универсальный методический шаблон дисциплины

Этот шаблон рекомендуется использовать для создания любого курса в MatrixGin.

### JSON-структура описания дисциплины (Manifest)

```json
{
  "discipline_meta": {
    "title": "Название дисциплины (Action-oriented)",
    "target_audience": ["Роль 1", "Роль 2"],
    "business_goal": "Какой KPI улучшаем?",
    "learning_objective": "Что сотрудник сможет СДЕЛАТЬ после курса?",
    "duration_minutes": 60
  },
  "modules": [
    {
      "order": 1,
      "title": "Проблематика и Контекст",
      "type": "motivation",
      "content": "Кейс провала / Статистика потерь / Выгода сотрудника"
    },
    {
      "order": 2,
      "title": "Алгоритм действия (Theory)",
      "type": "microlearning",
      "content": "Схема 1-2-3 / Чек-лист / Видео-демонстрация (3 мин)"
    },
    {
      "order": 3,
      "title": "Отработка (Practice)",
      "type": "simulation",
      "content": [
        {
          "type": "case_study",
          "scenario": "Клиент возражает 'Дорого'...",
          "options": ["Ответ А", "Ответ Б", "Ответ В"],
          "feedback_correct": "Отлично! Это работает, потому что...",
          "feedback_incorrect": "Рискованно. Вы звучите агрессивно."
        }
      ]
    },
    {
      "order": 4,
      "title": "Проверка боем (Assessment)",
      "type": "exam",
      "content": "Итоговый тест / Задание для выполнения на рабочем месте"
    }
  ]
}
```

### Чек-лист проверки качества материала

*   [ ] **Целеполагание:** Понятно ли сотруднику, зачем ему это тратить время?
*   [ ] **Размер:** Разбит ли контент на куски по 5-10 минут?
*   [ ] **Язык:** Написано ли это живым языком, без канцеляризмов?
*   [ ] **Практика:** Есть ли возможность ошибиться в безопасной среде (тесте/кейсе) до выхода к клиенту?
*   [ ] **Визуал:** Помогают ли картинки пониманию или просто украшают?
*   [ ] **Call to Action:** Есть ли в конце призыв применить навык сегодня?
