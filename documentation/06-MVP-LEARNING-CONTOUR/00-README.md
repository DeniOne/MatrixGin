# MVP Learning Contour — README (v1.1)

## 0. Назначение документа

Данный каталог `06-MVP-LEARNING-CONTOUR` фиксирует первый вводимый в эксплуатацию контур MatrixGin, ориентированный на обучение персонала, наставничество и защиту целостности образовательного процесса.

## 1. Цель MVP-контура

1.  Запустить **Корпоративный Университет** как полноценную экосистему знаний.
2.  Использовать **Dual-Interface UX**: Telegram для оперативного доступа и Web UI для глубокого обучения и управления.
3.  Ввести систему **Наставничества** с институтом акредитованных тренеров.
4.  Обеспечить **Целостность Обучения** через Anti-Fraud контур и аналитику без карательных функций.

## 2. Что входит в MVP (IN SCOPE)

### 2.1 Бизнес-контур
- **Регистрация сотрудника (Onboarding)**: Base-First Admission Gate (принятие Базы) + анкета через HR-Service.
- **Обучение**: Курсы, версионированные модули, Quiz-движок.
- **Наставничество**: Аккредитация тренеров (Grade-based), планы развития, 60-дневная пробация.
- **Аналитика**: Прозрачные дашборды эффективности обучения (Read-only).
- **Безопасность**: Детекция аномалий (Anti-Fraud) и ручной аудит сертификатов.

### 2.2 Технические модули
- **07 — Telegram Bot**: Уведомления, быстрый статус, микро-обучение.
- **13 — Corporate University**: Ядро обучения, Quiz Engine, Anti-Fraud Engine.
- **08 — MatrixCoin Economy**: Начисления за квалификацию (MC).
- **Web UI**: Полнофункциональный плеер курсов и Dashboards (Admin/HR/Trainer).
- **Infrastructure**: Event Registry & Audit Log (неизменяемость решений).

## 3. Что НЕ входит в MVP (OUT OF SCOPE)

### 3.1 Управление и контроль
- **Автоматические санкции**: Система ЗАПРЕЩАЕТ автоматическую блокировку за фрод. Только Human-in-the-loop.
- **Зарплатные механизмы**: Прямая привязка обучения к выплатам (только косвенно через грейд).
- **ERP-модули**: Склад, бюджеты и дебет/кредит.

## 4. Архитектурные Ограничения

### 4.1 Единство Интерфейсов
Сотрудник может начать обучение в Telegram и продолжить в Web. Состояние синхронизируется через Event Store.

### 4.2 Безопасность (Anti-Fraud)
В MVP включена детекция фрода. 
- Движок: Read-only анализатор.
- Мутация: Только через Audit Log с обязательным комментарием.
