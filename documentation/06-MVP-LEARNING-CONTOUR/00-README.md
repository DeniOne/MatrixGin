MVP Learning Contour — README
0. Назначение документа

Данный каталог 06-MVP-LEARNING-CONTOUR фиксирует первый вводимый в эксплуатацию контур MatrixGin, ориентированный на:

обучение персонала,

этическую мотивацию,

формирование культуры участия,

быстрый прикладной эффект без развёртывания всей системы.

Это не временная заглушка, а осознанно выделенный MVP-контур с чёткими границами и ограничениями.

1. Цель MVP-контура

Цель MVP:

Запустить Корпоративный Университет как живой инструмент обучения.

Использовать Telegram как единственный операционный интерфейс для сотрудников.

Ввести MatrixCoin (MC) как язык признания и фиксации участия.

Получить быстрый цикл обратной связи без внедрения ERP, KPI и управленческой аналитики.

Ключевой принцип:

Сначала ценность и участие — потом контроль и аналитика.

2. Что входит в MVP (IN SCOPE)
2.1 Бизнес-контур

В рамках MVP РАЗРЕШЕНО:

Корпоративное обучение (курсы, модули, задания)

Прохождение обучения сотрудниками

Фиксация факта участия и завершения

Начисление MatrixCoin (MC) за:

обучение,

активное участие,

вклад в культуру

Обратная связь сотруднику через Telegram

2.2 Технические модули

В MVP используются только следующие модули:

**Основные модули:**

07 — Telegram Bot (основной интерфейс)

13 — Corporate University (обучение)

08 — MatrixCoin Economy (MVP-режим)

**Критичные инфраструктурные модули:**

33 — Personnel HR Records (регистрация сотрудников, личные дела)

29 — Library & Archive (архивирование документов, 75-летнее хранение)

Базовая инфраструктура:

Auth

Employees

Event / Audit Log

**Важно:** Module 33 и Module 29 являются неотъемлемой частью MVP, так как:

- Module 33 обеспечивает регистрацию сотрудников в системе и создание личных дел
- Module 29 обеспечивает долгосрочное хранение документов (юридическое требование)
- Интеграция 33 ↔ 29 автоматически архивирует личные дела уволенных сотрудников

Все остальные модули считаются неактивными.

4. Роль AI в MVP (критически важно)
4.1 Разделение AI-контуров

В системе MatrixGin существует принципиальное разделение:

AI Core / Governance — архитектурный и нормативный слой

AI Interface / Runtime — прикладной слой взаимодействия

4.2 Что разрешено в MVP

В MVP РАЗРЕШЕНО использовать:

ai_mg_chat

intents

UX maps

error maps

schemas

AI в MVP:

отвечает,

объясняет,

сопровождает обучение,

помогает ориентироваться.

4.3 Что запрещено в MVP

В MVP ЗАПРЕЩЕНО:

Использовать 03-ai (AI Core) напрямую

Давать AI право:

оценивать людей,

начислять санкции,

принимать управленческие решения

Использовать AI для скрытого контроля

Использовать данные сотрудников для обучения внешних моделей

AI в MVP — ассистент и проводник, а не судья.

5. Telegram как единственный интерфейс

На этапе MVP:

❌ Нет сложного Web UI для сотрудников

❌ Нет дашбордов

❌ Нет внутренних порталов

✅ Telegram — единая точка входа

✅ Простые сценарии:

onboarding

обучение

задания

уведомления

обратная связь

Это осознанное UX-решение, а не ограничение ресурсов.

6. MatrixCoin в MVP

MatrixCoin в рамках MVP:

не валюта

не KPI

не инструмент давления

MC = этический маркер участия.

Назначение MC:

зафиксировать вклад,

поддержать мотивацию,

усилить вовлечённость.

Подробные правила описываются в: 05-MATRIXCOIN-RULES-MVP.md
