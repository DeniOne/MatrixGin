# MATRIXCOIN-RULES-MVP — Learning Contour (v1.1)

## 0. Назначение документа

Данный документ фиксирует правила использования MatrixCoin (MC) в рамках MVP Learning Contour v1.1, обеспечивая баланс между мотивацией через достижения и защитой от манипуляций.

## 1. Каноническое определение MatrixCoin

**MatrixCoin (MC) в v1.1 — это:**
> Этический маркер квалификации и активного участия, подтверждённый каноническими событиями системы (University Events).

## 2. Разрешённые основания для начисления

В v1.1 разрешено автоматическое начисление MC за:
- **Завершение курсов**: Успешное прохождение Quiz Engine с баллом выше порога.
- **Наставничество**: Успешное завершение Mentorship Period (начисляется и наставнику, и подопечному).
- **Квалификацию**: Повышение грейда через Qualification Engine.

## 3. Защита и Антифрод

- **Event-Driven**: Начисления происходят только на основе проверенных событий в Event Registry.
- **Audit Trace**: Каждое начисление за обучение должно иметь ссылку на ID события (напр., `QUIZ_COMPLETED`).
- **Suspension**: В случае подозрения во фроде (сигнал `Anti-Fraud Engine`), начисление MC блокируется до ручного аудита в Security Dashboard.

## 4. Этические ограничения
- **No Leaderboards**: Публичные рейтинги сотрудников по количеству MC запрещены.
- **Privacy**: Баланс MC виден только владельцу, HR и наставнику.

- **Статус**: Active / Binding
- **Контур**: MVP Learning Contour
