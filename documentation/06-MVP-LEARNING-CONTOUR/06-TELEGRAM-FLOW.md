# TELEGRAM-FLOW — MVP Learning Contour (v1.1)

## 0. Назначение документа

Данный документ описывает роль **Telegram Bot** в рамках **Dual-Interface UX** MatrixGin. Бот является операционным шлюзом и хабом уведомлений, дополняющим Web UI.

## 1. Общие принципы

1.  Telegram — **операционный интерфейс для ежедневных микро-действий**.
2.  Бот интегрирован с Web UI: учебный прогресс и состояние сессии синхронны.
3.  Принцип: **Бот нотифицирует и направляет, Web UI глубоко погружает.**

## 2. Ключевые сценарии (Flows)

### 2.1 Оперативный доступ
- Сотрудник получает в боте уведомление о новом курсе.
- Бот предоставляет `Deep Link` для мгновенного перехода в Web UI к конкретному модулю.

### 2.2 Микро-обучение (Light-path)
- Просмотр коротких текстовых материалов и чек-листов прямо в Telegram-интерфейсе.
- Подтверждение ознакомления без перехода во внешние системы.

### 2.3 Аттестационный Handover
- Бот напоминает о необходимости пройти Quiz.
- Для прохождения комплексных тестов (Quiz Engine v1.1) бот пересылает пользователя в Web UI.
- Результат прохождения (Passed/Failed) возвращается в бот в виде поздравительной открытки или рекомендации повторить.

### 2.4 Наставничество (Trainer View)
- Тренеры получают уведомления о новых подопечных (Mentorship Started).
- Бот позволяет быстро подтвердить встречу или просмотреть статус пробации.
