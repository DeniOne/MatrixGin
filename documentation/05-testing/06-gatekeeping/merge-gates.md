A change MAY be merged only if:

1. Strategic invariants are not violated.
2. Architectural boundaries are respected.
3. No contract is broken.
4. Relevant functional scenarios pass.
5. Required technical tests pass.
6. Change is documented.

Failing any gate blocks merge regardless of urgency.


### Mandatory Evidence

- Updated documents (if logic changed)
- Tests or validation artifacts
- Clear description of intent
