# DEVELOPMENT-CHECKLIST.md

## 25 — Emotional Passport

### Назначение чеклиста

Данный документ определяет **поэтапную, этичную и безопасную реализацию** модуля Emotional Passport.

Ключевой принцип:

> **Доверие важнее функциональности.**

Никакая автоматизация не допускается до формирования доверительного MVP.

---

## PHASE 0 — Guardrails & Ethics (ОБЯЗАТЕЛЬНО)

* [ ] Подтвердить, что Emotional Passport:

  * не используется для оценки эффективности
  * не используется для HR-решений
  * не используется для санкций
* [ ] Зафиксировать уровни видимости данных:

  * Private (только сотрудник)
  * AI-read-only
  * Manager-context-only
* [ ] Реализовать принцип: *данные принадлежат человеку*
* [ ] Исключить любые сравнительные или рейтинговые UI-паттерны

---

## PHASE 1 — MVP (Self-Reflection First)

### Цель

Дать человеку безопасный инструмент самопознания без давления.

### Функциональность

* [ ] Базовый Emotional Passport Profile
* [ ] Слой 1 — Источники энергии
* [ ] Слой 2 — Триггеры выгорания
* [ ] Слой 3 — Состояния потока
* [ ] Слой 4 — Интересы и резонансы

### Требования

* [ ] Все поля добровольные
* [ ] Нет обязательного заполнения
* [ ] Нет «завершённости» профиля
* [ ] UI = дневник / зеркало, не анкета

---

## PHASE 2 — Gentle Dynamics

### Цель

Сделать паспорт живым, но не навязчивым.

* [ ] Микро-вопросы (не чаще заданного лимита)
* [ ] Возможность редактирования и удаления данных
* [ ] История изменений (видна только сотруднику)

---

## PHASE 3 — AI Reflection (ОСТОРОЖНО)

### Цель

Использовать AI как зеркало, не как интерпретатор.

* [ ] AI может:

  * задавать уточняющие вопросы
  * отражать паттерны словами пользователя
* [ ] AI НЕ может:

  * делать выводы
  * давать советы без запроса
  * интерпретировать морально

---

## PHASE 4 — Integration Readiness

* [ ] Связь с State of Being Engine (read-only)
* [ ] Связь с Interest Gravity Engine
* [ ] Подготовка данных для Joy → Value Converter (этически ограниченно)

---

## PHASE 5 — Validation

* [ ] Тест доверия (qualitative feedback)
* [ ] Проверка отсутствия побочных эффектов
* [ ] Подтверждение соответствия философии MatrixGin

---

## STOP CONDITIONS (ЖЁСТКО)

Разработка ОСТАНАВЛИВАЕТСЯ, если:

* появляется давление на пользователя
* возникают попытки использования данных против человека
* теряется добровольность
