1. СМЫСЛ И НАЗНАЧЕНИЕ МОДУЛЯ
1.1. Что такое Library & Archive в MatrixGin

Library & Archive — это память MatrixGin.

Модуль отвечает за:

фиксацию знаний, решений и норм

сохранение истории без возможности переписывания

обеспечение единого источника истины

юридическую, смысловую и этическую воспроизводимость

Library & Archive — не файловое хранилище,
не СЭД,
не система обучения.

Это инфраструктура ответственности.

1.2. Какую человеческую ценность усиливает модуль

Модуль усиливает:

осознанность действий

доверие к системе

предсказуемость последствий

уважение к прошлым решениям

Library & Archive не контролирует человека,
а фиксирует след его действий,
делая систему честной по отношению к людям и к самой себе.

2. ФИЛОСОФСКИЕ И ЭТИЧЕСКИЕ ИНВАРИАНТЫ
2.1. Базовые принципы (НЕИЗМЕНЯЕМЫЕ)

Документ ≠ файл
Документ = смысл + контекст + ответственность + история.

Прошлое нельзя переписать
Любое решение остаётся зафиксированным.

Нет удаления — есть судьба
Документы:

активируются

архивируются

уничтожаются только по регламенту
но не «исчезают».

Источник истины всегда один
Другие модули могут ссылаться, но не дублировать истину.

Этика выше удобства
Если удобство требует потери следа — удобство запрещено.

2.2. Роль AI (жёсткое ограничение)

AI в Module 29:

❌ не создаёт документы

❌ не изменяет версии

❌ не принимает решений о статусе или сроках хранения

❌ не уничтожает документы

✅ читает утверждённые версии

✅ использует документы как контекст (RAG, reasoning)

✅ предлагает классификацию и связи (advisory)

AI никогда не является источником истины.

5. КАНОНИЧЕСКАЯ МОДЕЛЬ ДОКУМЕНТА
5.1. Тип документа

Тип документа определяет:

юридический статус

уровень доступа

срок хранения

допустимые действия

Типы документов регистрируются через System Registry
и не создаются произвольно.

5.2. Жизненный цикл документа

Канонический FSM:

draft → active → archived → destroyed


draft — не является источником истины

active — действующая версия

archived — выведен из оборота, но сохранён

destroyed — уничтожен по регламенту с актом

5.3. Версионирование

каждая версия неизменяема

используется semantic versioning

новая версия ≠ перезапись старой

ссылки всегда указывают: document_id + version

6. ВЗАИМОДЕЙСТВИЕ С ДРУГИМИ МОДУЛЯМИ
6.1. Corporate University

University никогда не хранит оригиналы

учебные материалы = ссылки на Library

курсы фиксируют:

какой документ

какую версию

в каком контексте

6.2. Legal / Compliance

Library хранит юридически значимые документы

retention policies задаются Legal

уничтожение возможно только:

по сроку

по основанию

с audit trail

6.3. AI Core / RAG

Library — основной источник контекста

AI читает только утверждённые версии

AI не может ссылаться на draft

все AI-ответы воспроизводимы через версию документа

7. ЗАПРЕЩЁННЫЕ СЦЕНАРИИ (КРАСНЫЕ ЛИНИИ)

Модуль считается архитектурно сломанным, если:
