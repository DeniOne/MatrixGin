# 04-before-production.md
## Before Production — Final Readiness Checklist (MC / GMC)

**Назначение:** Последняя проверка перед включением MatrixCoin Economy в продакшене  
**Принцип:** Лучше не запустить, чем запустить неправильно  
**Приоритет:** CANON → Audit → Operations → Реализация

---

## 0. Условия допуска (GATE)

Продакшн **РАЗРЕШЁН ТОЛЬКО ЕСЛИ** выполнены все пункты ниже:

- [ ] CANON (00-CANON/01–09) принят и LOCKED
- [ ] `MODULE-08-CANON.md` утверждён
- [ ] `MODULE-08-DOMAIN-LOCK.md` активен
- [ ] Нет незакрытых критических инцидентов
- [ ] Назначен единый владелец домена (человек)
- [ ] Есть подтверждённый план rollback

❌ Если хотя бы один пункт не выполнен — **СТОП**

---

## 1. Проверка документации (ОБЯЗАТЕЛЬНО)

### 1.1. Domain Rulebook (`01-REGULATIONS/`)

Проверь наличие и актуальность:

- [ ] `01-auction-mechanics.md`
- [ ] `02-mc-lifecycle.md`
- [ ] `03-gmc-lifecycle.md`
- [ ] `04-charging-and-spending.md`
- [ ] `05-edge-cases.md`

Все документы:
- [ ] согласованы между собой
- [ ] не противоречат CANON
- [ ] понятны без автора

---

### 1.2. Operational Checklists (`05-CHECKLISTS/`)

Проверь:

- [ ] `02-how-to-launch-mc.md`
- [ ] `03-incident-response.md`
- [ ] `04-before-production.md` (этот файл)
- [ ] `05-rollback.md` (должен быть готов)

---

## 2. Проверка системы (ТЕХНИЧЕСКИ)

- [ ] Audit работает и пишет события
- [ ] События append-only
- [ ] Нет ручных операций с балансами
- [ ] Нет автоматического арбитража
- [ ] Нет фоновых автодействий без человека
- [ ] AI работает в read-only / advisory-only

---

## 3. Проверка MC (КРИТИЧЕСКОЕ)

- [ ] Начисление MC привязано к действиям
- [ ] Нет начислений «за период»
- [ ] Нет начислений «за KPI»
- [ ] Есть лимиты начисления
- [ ] Есть kill-switch начислений

---

## 4. Проверка аукционов GMC

- [ ] Аукционы открываются вручную
- [ ] Нет параллельных конфликтующих аукционов
- [ ] Ставки не списывают MC сразу
- [ ] GMC не признаётся автоматически
- [ ] Победитель определяется по audit

---

## 5. Проверка коммуникации

Проверь тексты:

- [ ] Нет слов «заработал», «награда», «штраф»
- [ ] Нет сравнения людей
- [ ] Нет обещаний компенсаций
- [ ] Нет формулировок «система решила»

---

## 6. Проверка людей (НЕ ТЕХНИКА)

- [ ] Все ответственные понимают MC ≠ деньги
- [ ] Все понимают GMC ≠ награда
- [ ] Есть человек, который **может остановить систему**
- [ ] Нет давления «надо запускать срочно»

---

## 7. Dry Run (ОБЯЗАТЕЛЬНО)

Перед реальным запуском:

- [ ] Провести тестовый аукцион
- [ ] Провести тестовое начисление MC
- [ ] Пройти сценарий сбоя
- [ ] Проверить rollback (без фактического отката)

Если dry run выявил проблему — **НЕ ЗАПУСКАТЬ**

---

## 8. Финальное решение

Ответственный человек должен ответить:

- [ ] Я понимаю систему
- [ ] Я принимаю риски
- [ ] Я готов остановить запуск при сомнениях

Только после этого допускается GO LIVE.

---

## 9. Краткий чек (НА ОДИН ЭКРАН)

1. Документы есть  
2. Audit жив  
3. Автоматики нет  
4. Kill-switch есть  
5. Люди понимают  
6. Dry run пройден  

---

## 10. Главное правило

> **Продакшн — это не момент.  
> Это ответственность.**
