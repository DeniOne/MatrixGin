## 3. Data Model Constraints (без схем)

### 3.1 Общие принципы

- [ ] Нет сущностей с финансовым смыслом
- [ ] Нет полей, указывающих денежную стоимость
- [ ] Нет автоматических триггеров начисления ценности
- [ ] Нет вычисляемых «оптимальных путей»

### 3.2 MC

- [ ] MC хранится как **временное состояние**, а не накопительный актив
- [ ] MC не является мерой производительности
- [ ] MC не используется в аналитике эффективности человека

### 3.3 GMC

- [ ] GMC хранится отдельно от MC
- [ ] GMC не участвует в расчётах
- [ ] GMC не может быть списан автоматически

---

## 4. Event & Flow Restrictions

- [ ] Нет фоновых автоматических процессов начисления
- [ ] Все изменения состояния требуют **явного человеческого действия**
- [ ] Любой переход состояния логируем
- [ ] Отсутствуют циклы самоподдерживающегося роста

---

## 5. Store & Auction Implementation Guard

### 5.1 Store

- [ ] Нет ценников
- [ ] Нет корзин в классическом понимании
- [ ] Нет механик «чем больше — тем выгоднее»
- [ ] Store не стимулирует накопление MC

### 5.2 Auction

- [ ] Auction активируется только как событие
- [ ] Auction не работает в постоянном режиме
- [ ] Auction не гарантирует результат
- [ ] Auction не создаёт повторяемую стратегию участия

---

## 6. MC Lifecycle Enforcement

- [ ] Реализован механизм угасания MC
- [ ] Угасание не является наказанием
- [ ] Нет «спасательных» механик удержания MC
- [ ] Поведение пользователя важнее сохранения чисел

---

## 7. AI Integration Guard

- [ ] AI не имеет write-доступа
- [ ] AI не может инициировать события
- [ ] AI не может вызывать экономические операции
- [ ] AI не имеет доступа к персональным данным
- [ ] AI получает только агрегированные снапшоты
- [ ] AI рекомендации всегда маркированы как advisory

---

## 8. UX & Presentation Restrictions

- [ ] UX не использует финансовую терминологию
- [ ] UX не использует геймификацию накопления
- [ ] UX не стимулирует сравнение людей по MC/GMC
- [ ] Отсутствуют лидерборды по MC
- [ ] Отсутствуют визуальные триггеры «богатства»

---

## 9. Logging & Audit

- [ ] Все изменения состояния логируются
- [ ] Логи доступны для ручного аудита
- [ ] Отсутствуют скрытые изменения состояния
- [ ] Любая аномалия = остановка использования модуля

---

## 10. Forbidden Patterns (чёрный список)

Реализация считается **НЕДОПУСТИМОЙ**, если присутствует хотя бы один пункт:

- [ ] Автоматическое начисление ценности
- [ ] Прямая или косвенная монетизация MC / GMC
- [ ] AI как активный участник экономики
- [ ] Привязка MC к KPI или эффективности
- [ ] Механики фарма, оптимизации, гринда
- [ ] Давление на поведение через экономику
