I. НОРМАТИВНАЯ ЧАСТЬ (КАНОН v8)
1. Назначение MC в системе MatrixGin v8

MC (Matrix Coin) является операционной, сгораемой единицей вовлечённости, а не:

заработной платой;

бонусом;

компенсацией;

стратегическим активом.

MC используется исключительно для:

краткосрочной стимуляции участия в экосистеме;

поддержки поведения, выходящего за минимально ожидаемую роль;

формирования операционной динамики.

MC не накапливается как ценность и не гарантирует доступа к GMC.

Любое начисление MC допускается только при наличии зафиксированного условия в настоящем Реестре.

2. Каноническая структура записи условия MC (обязательная)

Каждое условие начисления MC обязано иметь строго следующую структуру.
Расширение, сокращение или интерпретация полей запрещены.

2.1. condition_id

Формат: MC-XXX

Уникальный идентификатор условия

Семантические или «говорящие» ID запрещены

2.2. action_definition

Формализованное описание действия

Формат: глагол + объект + контекст

Без оценочных и эмоциональных формулировок

2.3. completion_criteria

Полный список проверяемых критериев

Типы критериев:

бинарные (да / нет)

количественные (≥ / ≤ / =)

2.4. data_source

Допустимые источники данных:

Tasks

University

Ops / Production

Kaizen

Analytics (derived, non-raw)

2.5. impact_level

Ровно одно значение:

micro

local

significant

systemic

Примечание: systemic не означает стратегический актив и не ведёт к GMC.

2.6. mc_range

Формат: [min_mc ; max_mc]

Фиксированные значения запрещены

Итоговое значение определяется алгоритмом

2.7. verification_type

Допустимые значения:

automatic

validated

peer_confirmed

2.8. anti_duplication_flags

cooldown

max_per_period

exclusive_with

2.9. hard_exclusions

Обязательные флаги:

not_job_duty = true

not_kpi_based = true

not_manual_reward = true

Если любой флаг ≠ true, условие считается недопустимым.

draft

deprecated

2.11. version

Версия условия

3. Классификация типов условий MC (v8)

В системе MatrixGin v8 допускаются ТОЛЬКО ТРИ типа условий MC.

TYPE A — RESULT_BASED

Смысл:
MC начисляется за завершённый результат, выходящий за минимально ожидаемую роль.

Ключевые признаки:

результат зафиксирован;

не является должностной обязанностью;

имеет чёткий факт завершения.

Допустимые impact_level:
local, significant

Запрещено:

«хорошо выполнил работу»

«выполнил план»

«отработал смену»

TYPE B — CONTRIBUTION_BASED

Смысл:
MC начисляется за реализованный вклад в улучшение системы, а не за идею.

Ключевые признаки:

вклад реализован;

эффект измерим или зафиксирован;

вклад не является разовой помощью.

Допустимые impact_level:
local, significant, systemic

Запрещено:

предложения без внедрения;

участие без эффекта;

обсуждение без реализации.

TYPE C — DEVELOPMENT_BASED

Смысл:
MC начисляется за осознанное усложнение роли через применённое развитие.

Ключевые признаки:

обучение завершено;

применение зафиксировано;

развитие привело к новому уровню действий.

Допустимые impact_level:
micro, local

Запрещено:

MC за просмотр курсов;

MC за формальное тестирование;

MC за обучение без применения.

4. Запрещённые типы условий MC

Следующие типы условий НЕ МОГУТ СУЩЕСТВОВАТЬ в системе v8:

BEHAVIOR_BASED

EMOTIONAL_BASED

KPI_BASED

ROLE_BASED

TIME_BASED

LOYALTY_BASED

Любая попытка внедрения таких условий считается нарушением канона v8.

5. Общие системные запреты

В системе MatrixGin v8:

❌ нет ручного начисления MC;

❌ нет начислений «по усмотрению»;

❌ нет MC за должностные обязанности;

❌ нет автоматического начисления без верификации;

❌ нет гарантированных MC.
