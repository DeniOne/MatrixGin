## 3. СПИСОК ТОВАРОВ

### 3.1 Отображение товаров
Товары отображаются в виде списка карточек.

Каждая карточка содержит:
- Название товара
- Краткое описание
- Цена в MC
- Ограничения (если есть)
- Кнопку действия

Пример карточки:
Дополнительный выходной
Цена: 500 MC

Ограничения:
– не более 1 в месяц
– нельзя в пиковые дни

[ КУПИТЬ ]

yaml
Копировать код

---

### 3.2 Состояния товара

#### Активный товар
- Кнопка **КУПИТЬ** активна
- Цена отображается обычным текстом

#### Недостаточно MC
- Кнопка **КУПИТЬ** disabled
- Под кнопкой текст:
Недостаточно MC

markdown
Копировать код

#### Ограничение по правилам
- Кнопка disabled
- Под кнопкой текст:
Недоступно по условиям

markdown
Копировать код

#### Неактивный товар
- Карточка отображается серой
- Кнопка отсутствует
- Текст:
Временно недоступно

yaml
Копировать код

## 4. ПОКУПКА ТОВАРА (BUY FLOW)

### 4.1 Нажатие кнопки «КУПИТЬ»
При нажатии:
- отправляется запрос `POST /api/store/purchase`
- кнопка переходит в состояние loading
- повторное нажатие блокируется

---

### 4.2 Успешная покупка

После успешного ответа API:
- показывается success-уведомление:
Покупка успешна

yaml
Копировать код
- MC баланс обновляется
- товар становится недоступным, если есть лимит
- запись появляется в истории транзакций

## 6. ТРЕБОВАНИЯ К UX (ОБЯЗАТЕЛЬНО)

- Пользователь **всегда видит**, почему он не может купить товар
- Нет «молчаливых» ошибок
- Баланс MC обновляется сразу после покупки
- Нельзя купить товар повторно, если ограничение запрещает

---

## 7. ЧТО ПОЛЬЗОВАТЕЛЬ ПОНИМАЕТ

После использования Store пользователь понимает:
- что такое MC
- что MC можно тратить
- за что он платит
- что происходит после покупки

---

## 8. OUT OF SCOPE (НЕ ДЕЛАЕМ В ЭТОМ ШАГЕ)

- Аукционы
- GMC-покупки
- Admin-управление магазином
- Массовые покупки

---

## 9. ACCEPTANCE CRITERIA

Store считается реализованным, если:
- пользователь видит Store Page
- пользователь видит список товаров
- пользователь может купить товар
- пользователь видит результат покупки
- пользователь понимает ошибки
